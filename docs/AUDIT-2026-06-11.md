# DC Instructor — Repo Audit & Improvement Plan (2026-06-11)

Audited at `main` commit `370e473`. Fixes applied on branch `v1.2` (commit `d2ba9bf`).

## Executive Summary

**Grade: B−** (was C+ before v1.2 fixes). The product core is genuinely good: a small, readable React + Netlify Functions app with a thoughtful two-layer PHI redaction design and keys correctly kept server-side. The grade is held down by three things: (1) the usage cap is the *only* abuse control on public endpoints that spend Anthropic/Stability API credits, and it fails open — and per `metrics/snapshots.md` the Supabase project backing it has been unreachable for 11 consecutive snapshots, so the cap is currently NOT enforced in production; (2) until v1.2 there were zero tests and no CI on a tool that generates medication dosing text; (3) telemetry/observability is broken, so there is no usage data from the entire trial period — a problem for an investor demo that wants traction numbers. Top risks: unbounded API spend on open endpoints, PHI redaction regressions with no eval harness, and dead telemetry. Top opportunities: the `knowledge/` RAG pipeline (grounded generation is the defensible story), restoring metrics before the demo, and the now-passing CI gate as a foundation for safe iteration.

## Repo Map

**Purpose:** Plain-language ED discharge instruction generator for clinicians. Reading level 4th–10th grade or HL-1, any language, optional line-art illustration, PHI auto-redaction of pasted ED notes. Maturity: early beta / pilot (v0.1.0), single developer, ~500-generation trial.

**Stack:** React 18 + Vite 5 + Tailwind 3 (SPA) · Netlify Functions (ESM, web-standard `Request`/`Response`) · Anthropic Claude (Sonnet 4.5 generation, Haiku 4.5 PHI verify) · Stability AI (images) · Supabase (usage counter + ratings) · Python 3 / Chroma / sentence-transformers (`knowledge/` RAG pipeline, local-only DB).

**Flow:** `App.jsx` → client regex redaction (`src/lib/phiRedact.js`) → LLM verify (`/redact-verify`) → streaming generation (`/generate`, NDJSON) → optional `/generate-image` → rating modal every 5th generation → Supabase logging via service role.

| Area | What it is |
|---|---|
| `src/` | SPA: 1 page, 7 components, 4 lib modules (~600 LOC) |
| `netlify/functions/` | 4 serverless functions + shared `_lib.js` (~290 LOC) |
| `supabase/migrations/` | Schema: `generations`, `ratings`, `generation_count` view, RLS |
| `knowledge/` | MedlinePlus → Chroma ingestion, retrieval evals, clinician review queue (~2,150 LOC Python) |
| `library/` | **Empty** (`.gitkeep`) — target of the agent-built instruction library, not started |
| `metrics/snapshots.md` | Automated usage snapshots — all failed since launch |
| `CLAUDE.md` / `AGENTS.md` | Build-agent instructions (Claude / Codex) for the library workflow |

**Surprises:** (1) Eleven consecutive failed metrics snapshots — Supabase paused + missing env key — meaning zero usage data exists for the trial. (2) `library/` is empty despite two agent-instruction files describing it in detail. (3) The repo is really two projects: the shipped web app and a research-grade RAG pipeline that the app does not yet use.

## Audit Report

### Security

- **HIGH (fact, fixed in v1.2):** Usage cap fails open with no logging — `netlify/functions/_lib.js:20-22` returned `{ blocked: false }` on any Supabase error. Combined with the paused Supabase project, the 500-generation cap is not enforced **right now** in production. Anyone with the URL can spend unlimited Anthropic credits. v1.2 adds logging + `degraded` flag; fail-open behavior deliberately retained for demo availability — see Open Questions.
- **HIGH (fact, mitigated in v1.2):** No origin/abuse control on key-spending endpoints — `generate.js`, `generate-image.js`, `redact-verify.js` accepted any POST from anywhere. v1.2 adds a same-origin guard (`_lib.js:originAllowed`). This blocks cross-site browser abuse but **not** curl/scripts — real per-IP rate limiting still needed (Task M1-2).
- **MEDIUM (fact, fixed in v1.2):** Unvalidated prompt interpolation — `readingLevel` and `language` were interpolated raw into the system prompt (`generate.js:80` pre-fix), letting a crafted request inject system-level instructions. v1.2 validates `readingLevel` against an allowlist and caps/sanitizes `language` and all input lengths.
- **MEDIUM (fact):** PHI verify failure degrades silently for the user — `App.jsx:50-53`: if `/redact-verify` errors, the regex-only scrubbed note is still sent to Anthropic with only a badge-state change (`verified: false`). Defensible (layer 1 already ran), but the clinician gets no explicit "second check did not run" warning. Not fixed — UX decision.
- **LOW (fact):** No secrets in repo or git history (scanned all commits). `.env` properly ignored, keys server-side only, RLS posture in `001_initial_schema.sql` is correct (anon can only insert ratings / read the counter). Solid.
- **LOW (fact, fixed in v1.2):** No CSP/HSTS headers — added in `netlify.toml`.

### Correctness & code quality

- **MEDIUM (fact):** PHI redactor over-redacts clinical content — `phiRedact.js`: the ZIP pattern `\b\d{5}(?:-\d{4})?\b` matches any 5-digit number (a WBC of `12000` becomes `[ZIP]`), and the NAME bigram heuristic ate common phrases like "Urgent Care" and "Physical Therapy". v1.2 expands the whitelist (+22 bigrams) and adds 16 tests pinning behavior; the ZIP/lab-value collision is deliberately left privacy-first (over-redaction is the safe direction) — needs an eval corpus before loosening (Task M2-1).
- **MEDIUM (judgment):** No machine-readable structure in generation output — the model emits free text into a `<textarea>`; nothing validates the 6 required sections actually appear. A malformed generation fails silently. (Task M2-2.)
- **LOW (fact, fixed in v1.2):** Dead code — `App.jsx:90` checked for an `'expired'` reason the server can no longer return (gate removed in `cdd14ae`); `ratingCadence.js` had a redundant condition.
- **LOW (fact):** `App.jsx:74` sets `setCount(meta.count + 1)` from the pre-generation count — off-by-timing cosmetic only.
- Code quality overall is **healthy**: small files, consistent style, no god objects, sensible error handling at API boundaries (`api.js:5-14`), non-fatal image failure handling (`App.jsx:77-84`).

### Testing

- **HIGH (fact, fixed in v1.2):** Zero JS tests, no CI, for an app that outputs medication dosing. The PHI redactor — pure, high-stakes, regex-heavy — was completely unpinned; any regex tweak could silently leak PHI or destroy note quality. v1.2 adds vitest (16 tests over `phiRedact` + `ratingCadence`, all passing) and a GitHub Actions workflow (test + build on push/PR). Remaining gap: no function-level tests for `generate.js` validation and no PHI eval corpus (synthetic notes + expected redactions).
- **LOW (fact):** The Python side is better — `rag_smoke_test.py` and committed retrieval evals (`knowledge/evals/*.json`) exist, though they're manual, not CI-run.

### Operations & observability

- **CRITICAL (fact, not repo-fixable):** Telemetry has never worked — `metrics/snapshots.md` shows 11/11 failed snapshots: Supabase free-tier project paused, `SUPABASE_SERVICE_ROLE_KEY` missing from the snapshot environment, DNS to `noloieuagfigaqahspfi.supabase.co` failing. Consequences: usage counter shows "—", ratings are lost, the cap is unenforced, and **you have no traction data for the investor demo.** Requires dashboard action (unpause/upgrade Supabase, set env vars), not code.
- **MEDIUM (fact):** No linting/formatting enforcement (no ESLint/Prettier config). Low pain at current size; add before any second contributor.

### Documentation & dependencies

- **MEDIUM (fact, fixed in v1.2):** README contradicted code — documented the `LAUNCH_DATE` 30-day cutoff removed in `cdd14ae`; `.env.example` still listed `LAUNCH_DATE`. Both corrected.
- **LOW (judgment):** `CLAUDE.md` and `AGENTS.md` are near-duplicate instructions for an empty `library/` and reference `knowledge/db/` which is git-ignored and must be rebuilt locally — confusing for any newcomer (or agent) who takes them at face value.
- **LOW (fact):** Dependencies are current-ish and minimal (React 18.3, Vite 5.4.21, supabase-js 2.105). Vite 5 is EOL-adjacent (Vite 6/7 exist) — upgrade eventually, no known exploitable CVE for this usage. Lockfile committed. No license file (fine for a private repo, decide before any public release).

### Strengths (preserve these)

1. Correct secret architecture: all third-party keys behind Netlify Functions; verified nothing in bundle or git history.
2. The two-layer PHI design (deterministic regex first, LLM second, originals never leave the browser) is the right shape and a real differentiator.
3. Streaming UX done properly end-to-end (SSE → NDJSON → incremental render).
4. Supabase RLS is correctly restrictive.
5. The `knowledge/` pipeline is unusually rigorous for a solo project: versioned manifests, committed retrieval evals, v0-vs-v1 comparisons, an intent-aware reranker.
6. Honest UI: clinician-review disclaimer above output, QI-research notice in footer.

## Improvement Strategy

**Theme 1 — The app trusts infrastructure that is currently down.** The cap, counter, ratings, and metrics all assume a live Supabase; it is paused, and every failure path was silent. Target: every external dependency either works or fails *visibly* (logs + `degraded` flags — v1.2 ships the visibility; restoring Supabase is an ops task). Done = a metrics snapshot succeeds and the live counter renders a number.

**Theme 2 — Public endpoints with no abuse story.** Target: layered controls — origin guard (shipped), per-IP rate limit, input caps (shipped), validated enums (shipped). Done = a curl loop from one IP gets 429s and worst-case daily spend is bounded.

**Theme 3 — Safety-critical logic had no safety net.** Target: CI-gated tests for everything pure (shipped for redactor/cadence), plus a PHI eval corpus and an output-structure check. Done = CI red blocks merge; redactor recall measured against a synthetic corpus.

**Theme 4 — Two projects, one repo, not yet connected.** The RAG pipeline (the investor-grade moat: *grounded* discharge instructions) is unused by the live app, and `library/` is empty. Target: pick the demo narrative — either wire retrieval into `/generate` for the covered conditions or present the pipeline as the roadmap. Decide, don't drift.

**Deliberately NOT fixing:** TypeScript migration (repo is small and consistent; cost > payoff pre-validation); loosening the ZIP/lab-value over-redaction without an eval corpus (wrong risk direction); Vite 6 upgrade (no exploit, churn risk before a demo); ESLint (worth ~1h, but behind everything above); consolidating CLAUDE.md/AGENTS.md (cosmetic).

### Implementation sketches (top 3 remaining)

**1-1 Supabase restore (do first, ~30 min, all dashboard):** Unpause project → confirm `curl https://<ref>.supabase.co/rest/v1/generation_count` resolves → upgrade to a paid tier or add a keep-alive ping so it can't re-pause before the demo → add `SUPABASE_SERVICE_ROLE_KEY` + `*.supabase.co` network allowance to the Claude-Code-on-the-web snapshot environment → run one snapshot manually. Gotcha: a paused free-tier project can be auto-deleted after long inactivity — verify the data still exists *today*.

**1-2 Rate limiting:** Prefer Netlify's built-in rate limiting (zero code) if your plan has it; otherwise add `@upstash/ratelimit` in `_lib.js` keyed on `x-nf-client-connection-ip` with a fail-open-plus-log posture consistent with `checkLimits`. Apply in all three functions before the upstream fetch. Gotcha: don't rate-limit `/usage` (read-only, cheap); and remember serverless instances don't share memory — the limiter must be external (Redis) or platform-level.

**2-3 RAG wiring:** In `/generate`, before the Anthropic call, POST condition+level to a new retrieval path. Cheapest viable: since Chroma is local-only, run `ingest_medlineplus.py` offline, export top sections per covered condition to a static JSON the function can bundle (no Python in prod). Inject as a `GROUNDING:` block in the system prompt with "prefer this content for home care and return precautions." Gotcha: condition strings are free text — match against `ALLOWED_TOPICS` with a fuzzy/alias map and fall back to ungrounded silently.
